## Audit method

Each item was verified by:
1. Reading the export surface of the module the item references (`grep -n ^export <module>`).
2. Tracing every callsite of the load-bearing function via `grep -rn` across `src/`, `container/agent-runner/src/`.
3. Reading any boot-time wiring in `src/index.ts` to confirm the function is started/registered at process startup.
4. Confirming a test file exists in `src/__tests__/` or `src/permissions/__tests__/` for the behavior, and reading the test's docstring or first test case to confirm the surface under test matches the BACKLOG claim.

**Confidence calibration:**
- **HIGH** — read the full call chain end-to-end (export → caller → test) and saw matching evidence at each layer.
- **MEDIUM** — saw scaffolding + tests, but did not run the system or step through the runtime.
- **LOW** — saw declarations, no usage trace; or saw indirect evidence only.

This audit is HIGH-confidence on all 8 items. Each has at least 2 file:line evidence points showing both the wiring and a test case.
